## ⚠️ Point-in-time 局限(先读这个,结论的天花板)

**本回放用"今天(2026-06-10)的 PROVEN 名单"回放历史 = 幸存者偏差 + 前视。** 这 5 个 handle 之所以 PROVEN,正是因为它们历史 call_outcomes 命中率过了 Wilson 下界——拿它们的历史重放,命中率"好看"是**选择标准的同义反复**,不是发现。历史各时点的真实 PROVEN 名单无从重建(无榜单快照存档)。因此:**本报告只回答"这套跟单规则是否 sane(不破坏信号、不产生荒谬行为)",不构成 edge 证明。edge 的唯一验证是模拟盘前向实测。**

## 方法

入场 = `call_outcomes.entry_close`(喊单日后首交易日收盘,T+1 close,上游口径原样消费不重算);退出 = 持有 21 交易日(= 同行 `exit_close`);收益用上游 `fwd_return`/`abnormal_return`(对 SPY 超额)/`is_hit`(±0.5% 死区→不计)。规则按 spec §1 顺序:同日同 handle×ticker 去重 → 同日同票择优(wilson_lo→conviction→confidence)→ 一票一仓 → 7×24h 内任一 PROVEN bearish 冲突跳过 → 单 handle ≤5 并发。剔除回填长尾(call→entry 日历 gap ≥5 天,75 行——实为上游价格序列缺口致 entry 贴到 2024-06-03,非真实 T+1)。

## 规则行为观察(sane check 本体)

1. **规则没有破坏信号**:子集表现 ≥ 无约束基线(60.4% vs 54.8% hit;+6.0% vs +3.3% 超额)。差异不可读作"规则增益"(n=101,且 handle_cap 造成的入选≈"槽位空闲时先到先得",有运气成分),读作"无害"即可。
2. **handle_cap≤5 是绝对主导约束**(58% 候选被它挡掉)——证实 spec §5.3 容量错配:交易的是排队子集,**表现≠榜单全样本**,前向亦然。
3. **并发峰值 15、均值 6.5**:spec 的 N=10 总槽上限会在 ~6% 决策日(16/274)实际触发,本回放未建模 N=10——真实规则下成交略少于 108 笔。
4. **冲突跳过仅 2%**(86/4,161),与 Strat 实测 ~3% 一致,不是主约束。
5. **规则盲点(建议 v1 处理)**:GOOG(6 笔)与 GOOGL(4 笔)被当两票,一票一仓挡不住同公司双类股;成交集中在动量大票(TSLA 9、NVDA 6、GOOG+GOOGL 10),叠加 −48% 级单笔尾损,集中度风险真实存在(止损在本回放中**未建模**,见局限)。

## 局限(除 PIT 外)

①止损 −8% 与同博主翻空退出**无法用 call_outcomes 建模**(无价格路径)——真实规则的退出分布会不同,方向未知;②无 N=10 总槽;③$3 价格闸用 entry_close 近似决策时现价;④冲突窗口锚定 T_entry 19:30 UTC(15:30 ET 近似);⑤逐笔等权统计,非组合资金曲线;⑥活库快照,重跑数字会随上游评估推进微动。

## 结论

**Sane:是。** 规则管线在 22 个月历史上产生行为合理、可解释的交易流(~5 笔/月),各过滤器作用方向与设计意图一致,未发现规则相互打架或系统性丢单/荒谬成交。**Edge:未证明,本方法论上不可证明**(见 PIT 声明);Wilson 下界 0.507 仅勉强过 0.5,且全部统计建立在幸存名单上。**建议:按 spec 开模拟盘前向验证,以 2-4 周真实数据复议 handle_cap=5 与集中度(GOOG/GOOGL 归并、同主题敞口上限)。**
